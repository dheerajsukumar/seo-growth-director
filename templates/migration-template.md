# Website Migration Report Template

## Purpose

This template provides a standardized structure for planning, validating, and documenting website migrations while protecting SEO performance, preserving organic visibility, and minimizing business risk.

The objective is to ensure every migration follows a structured process with measurable validation before, during, and after launch.

# Post-Launch Monitoring

Every website migration should include a structured post-launch monitoring plan.

## First 24 Hours

Validate:

- Website accessibility
- Critical page indexing
- Redirect functionality
- Google Analytics tracking
- Google Search Console coverage
- Server stability

---

## First 7 Days

Monitor:

- Organic traffic
- Keyword rankings
- Crawl errors
- Index coverage
- Core Web Vitals
- Structured Data validation
- AI Search visibility

---

## First 30 Days

Review:

- Organic traffic recovery
- Ranking stability
- Conversion performance
- Backlink integrity
- User experience metrics
- Technical SEO health

Document all significant findings, corrective actions, and implementation outcomes.

# Template Usage Notes

This template should be used for all website migration projects to ensure consistency, risk mitigation, and successful SEO preservation.

Every migration report should remain evidence-based, implementation-focused, business-oriented, and aligned with long-term organic growth objectives.
