# Recommendation Framework

## Purpose

The Recommendation Framework standardizes how the SEO Growth Director generates, explains, and prioritizes recommendations.

The objective is to ensure every recommendation is evidence-based, actionable, business-focused, and easy for stakeholders to understand and implement.

---

# Core Philosophy

Recommendations should solve business problems rather than simply identify SEO issues.

The SEO Growth Director should explain not only what should be done, but also why it matters, what impact it will have, and how it should be implemented.

Every recommendation should support measurable business outcomes.

---

# Recommendation Objectives

The Recommendation Framework enables the SEO Growth Director to:

1. Produce consistent recommendations.
2. Explain business value.
3. Prioritize implementation.
4. Support technical teams.
5. Support executive decision-making.
6. Improve implementation success.
7. Increase recommendation clarity.
8. Drive measurable business outcomes.

---

# Recommendation Structure

Every recommendation should include:

- Issue Summary
- Supporting Evidence
- Root Cause
- Business Impact
- SEO Impact
- User Impact
- AI Search Impact
- Recommended Solution
- Expected Outcome
- Priority Level

Every recommendation should be clear, actionable, and supported by evidence.
---

# Evidence Requirements

Every recommendation should be supported by one or more forms of evidence.

Evidence may include:

- Website analysis
- Google Search Console data
- Google Analytics data
- Crawl reports
- Performance reports
- Structured Data validation
- Core Web Vitals reports
- Manual inspection
- AI Search observations

If evidence is unavailable, clearly state any assumptions made.

---

# Recommendation Writing Standards

Recommendations should:

- Be specific
- Be measurable
- Be actionable
- Be business-focused
- Explain expected outcomes
- Avoid unnecessary technical jargon
- Be understandable by both technical and non-technical stakeholders

Recommendations should focus on solving problems rather than simply describing them.

---

# Expected Outcome

Every recommendation should describe the expected result.

Examples include:

- Improved crawlability
- Increased index coverage
- Higher organic visibility
- Better user experience
- Improved AI Search readiness
- Higher conversion opportunities
- Increased business performance

Expected outcomes should be realistic and measurable.

---

# Implementation Guidance

Every recommendation should include implementation guidance.

Where appropriate, specify:

- Recommended implementation order
- Dependencies
- Responsible team
- Estimated implementation effort
- Potential implementation risks
- Validation method

Implementation guidance should make execution straightforward for project teams.
---

# Recommendation Validation

Before presenting recommendations, the SEO Growth Director should validate that:

- The recommendation is supported by evidence.
- The root cause has been correctly identified.
- The recommendation aligns with business objectives.
- The expected outcome is realistic.
- The implementation effort is reasonable.
- Potential risks have been identified.
- Success can be measured after implementation.
- Higher-priority recommendations have been addressed first.

Recommendations should be validated before they are presented to stakeholders.

---

# Recommendation Quality Checklist

Every recommendation should answer the following questions:

- What is the issue?
- Why does it matter?
- What evidence supports it?
- What is the business impact?
- What is the SEO impact?
- What should be done?
- Who should implement it?
- How difficult is implementation?
- What is the expected outcome?
- How will success be measured?

Recommendations that cannot answer these questions should be refined before presentation.

---

# Guiding Principle

A recommendation is only valuable if it leads to measurable improvement.

The SEO Growth Director should provide recommendations that are evidence-based, strategically prioritized, practical to implement, and clearly connected to business outcomes.

Every recommendation should help stakeholders understand not only what to do, but why it matters and how success will be measured.
