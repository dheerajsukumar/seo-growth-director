# Repository Map

The SEO Growth Director Skill is powered by the repository located at the project root.

## Repository Structure

### identity/

Defines:

- Role
- Behaviour
- Expertise
- Communication style

---

### knowledge/

Contains subject matter expertise for:

- Technical SEO
- Content Strategy
- Keyword Research
- Local SEO
- Ecommerce SEO
- AI Search
- Entity SEO
- Semantic SEO
- Website Migration

---

### workflows/

Defines execution processes including:

- Technical Audit
- Content Audit
- Keyword Research
- Competitor Analysis
- AI Search Assessment
- Executive Reporting

Always select the most appropriate workflow before responding.

---

### templates/

Provides structured output formats for:

- Audits
- Executive Reports
- Roadmaps
- AI Search Reports
- Monthly Reports

Use templates whenever producing structured deliverables.

---

### shared/

Contains reusable business frameworks including:

- Prioritization
- Risk Assessment
- Opportunity Evaluation
- Success Metrics

Apply these consistently across recommendations.

---

### examples/

Represents the expected quality standard.

Responses should meet or exceed the quality demonstrated in these examples.

---

### docs/

Contains architecture and supporting documentation.

Use for clarification when repository behaviour is uncertain.

---

## Navigation Principles

Always:

1. Identify the user's objective.
2. Select the correct workflow.
3. Load supporting knowledge.
4. Apply the appropriate template.
5. Produce an executive-ready response.

The repository is the authoritative source of knowledge. Avoid duplicating information that already exists there.
