# Technical SEO

## Purpose

Technical SEO forms the foundation of sustainable organic growth.

The purpose of this document is to provide the SEO Growth Director with deep domain knowledge about website architecture, crawling, indexing, rendering, performance, structured data, and technical implementation best practices.

This knowledge should be applied before recommending content expansion, authority building, or advanced optimization strategies.

---

# Core Philosophy

Technical SEO exists to make websites:

- Discoverable
- Crawlable
- Renderable
- Indexable
- Understandable
- Fast
- Secure
- Accessible

Strong technical foundations maximize the effectiveness of every other SEO initiative.

---

# Technical SEO Objectives

The SEO Growth Director evaluates websites according to these primary objectives:

1. Ensure search engines can discover every important page.
2. Ensure valuable pages can be indexed correctly.
3. Remove unnecessary crawling obstacles.
4. Improve rendering efficiency.
5. Optimize website architecture.
6. Improve internal linking.
7. Enhance page performance.
8. Implement structured data.
9. Improve mobile usability.
10. Maintain technical stability during website growth.

---

# Technical SEO Framework

The SEO Growth Director evaluates every website using the following technical pipeline.

```
Discovery
      ↓
Crawling
      ↓
Rendering
      ↓
Indexing
      ↓
Understanding
      ↓
Ranking
      ↓
User Experience
```

Technical problems should always be diagnosed by identifying where the breakdown occurs in this pipeline.

---

# Website Architecture

Website architecture defines how efficiently users and search engines can navigate a website.

The SEO Growth Director evaluates:

- Site hierarchy
- URL structure
- Navigation
- Internal linking
- Taxonomy
- Content depth
- Crawl depth
- Logical grouping of pages

Best Practices

- Keep important pages within three clicks.
- Maintain descriptive, human-readable URLs.
- Avoid orphan pages.
- Build clear topic clusters.
- Organize content according to search intent.
- Minimize unnecessary URL parameters.

Architecture should always support both users and search engines.

---

# Crawling

Search engines must successfully crawl a page before it can be indexed.

The SEO Growth Director evaluates:

- robots.txt
- XML Sitemap
- Crawl Budget
- Crawl Depth
- Response Codes
- Redirect Chains
- Redirect Loops
- Broken Links
- Canonical Signals

Common Crawl Issues

- Blocked resources
- Infinite URL spaces
- Duplicate URLs
- Incorrect canonicals
- Soft 404s
- Excessive redirects
- Broken internal links

Recommendations should always improve crawl efficiency before attempting ranking optimization.

---

# Rendering

Modern websites rely heavily on JavaScript.

Rendering determines whether search engines can access the actual page content.

The SEO Growth Director evaluates:

- Server Side Rendering (SSR)
- Client Side Rendering (CSR)
- Dynamic Rendering
- JavaScript execution
- Resource loading
- Lazy loading
- Hydration issues

Rendering failures frequently cause indexing problems despite successful crawling.

---

# Indexing

Only indexed pages can appear in search results.

The SEO Growth Director evaluates:

- Index Coverage
- Canonical URLs
- Duplicate Content
- Noindex Directives
- XML Sitemap Accuracy
- Parameter Handling
- HTTP Status Codes

Goals

- Maximize index quality
- Eliminate duplicate pages
- Reduce index bloat
- Maintain canonical consistency
- Ensure important pages remain indexable
