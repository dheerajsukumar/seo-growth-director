# Structured Data

## Purpose

The Structured Data knowledge base enables the SEO Growth Director to improve how search engines and AI systems understand website content through standardized schema markup.

The objective is to enhance semantic understanding, increase rich result eligibility, strengthen entity recognition, and improve visibility across traditional search engines and AI-powered search experiences.

---

# Core Philosophy

Structured Data provides explicit meaning to website content.

Rather than requiring search engines to infer information, structured data clearly communicates:

- What the content represents
- Which entities are involved
- How entities relate
- Which attributes describe each entity
- How content fits into the broader Knowledge Graph

Structured Data improves machine understanding without changing the visible content.

---

# Structured Data Objectives

The SEO Growth Director applies Structured Data to:

1. Improve semantic understanding.
2. Increase rich result eligibility.
3. Strengthen Knowledge Graph signals.
4. Improve entity recognition.
5. Support AI Search visibility.
6. Improve content discoverability.
7. Enhance search result presentation.
8. Strengthen trust signals.
9. Improve machine readability.
10. Build long-term semantic authority.

# Structured Data Framework

Every Structured Data implementation follows this framework.

```
Content
      ↓
Primary Entity
      ↓
Schema Selection
      ↓
Required Properties
      ↓
Validation
      ↓
Knowledge Graph
      ↓
Rich Results
      ↓
AI Understanding
```

The SEO Growth Director should always identify the primary entity before recommending schema markup.

# Schema Decision Framework

The SEO Growth Director should recommend schema based on the primary purpose of the page.

```
Is this page about a business?
        │
       Yes
        │
        ▼
Use Organization or LocalBusiness Schema

        │
       No
        │
        ▼
Is this page about a person?

        │
       Yes
        │
        ▼
Use Person Schema

        │
       No
        │
        ▼
Is this page an article?

        │
       Yes
        │
        ▼
Use Article Schema

        │
       No
        │
        ▼
Is this page selling a product?

        │
       Yes
        │
        ▼
Use Product Schema

        │
       No
        │
        ▼
Evaluate additional Schema.org types.
```

Every page should have a clearly identified primary schema type before considering secondary schema types.

---

# Schema Validation

Before recommending implementation, validate that:

- The schema matches the primary entity.
- Required properties are complete.
- Structured data reflects visible page content.
- URLs are valid and canonical.
- Images are accessible.
- Dates are correctly formatted.
- Organization information is consistent.
- Structured data passes Schema.org validation.

Invalid structured data reduces search engine confidence.

---

# Common Schema Mistakes

The SEO Growth Director should identify and avoid:

- Missing required properties
- Incorrect schema type selection
- Conflicting structured data
- Duplicate schema
- Fake review markup
- Hidden structured data
- Outdated schema properties
- Inconsistent entity information

Quality is more important than quantity when implementing schema.

# Structured Data Optimization Checklist

Before recommending Structured Data implementation, validate that:

- The primary entity has been identified.
- The correct Schema.org type has been selected.
- Required properties are complete.
- Optional properties add meaningful context.
- Structured data matches visible page content.
- Entity information is consistent across the website.
- Organization and author information are complete.
- Schema validation has been performed.
- Rich Result eligibility has been evaluated.
- AI Search understanding has been strengthened.

Structured Data should improve both machine understanding and user trust.
