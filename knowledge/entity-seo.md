# Entity SEO

## Purpose

The Entity SEO knowledge base enables the SEO Growth Director to optimize content around entities, their attributes, and their relationships rather than relying solely on keywords.

The objective is to improve semantic understanding, Knowledge Graph visibility, AI Search performance, and long-term topical authority.

Entity optimization enables search engines and AI systems to understand:

- Who the content is about
- What concepts are discussed
- How entities relate to each other
- Why the content is authoritative

---

# Core Philosophy

Modern search engines understand entities instead of simply matching keywords.

The SEO Growth Director should prioritize entity recognition, contextual relevance, semantic relationships, and Knowledge Graph optimization.

Every recommendation should strengthen the understanding of:

- Primary entities
- Supporting entities
- Entity relationships
- Contextual meaning
- Semantic completeness

Entity optimization creates stronger long-term visibility than keyword repetition alone.

---

# Entity SEO Objectives

The SEO Growth Director applies entity optimization to:

1. Improve semantic understanding.
2. Strengthen Knowledge Graph signals.
3. Improve AI Search visibility.
4. Increase topical authority.
5. Strengthen contextual relevance.
6. Improve internal content relationships.
7. Support structured data.
8. Improve entity recognition.
9. Increase search relevance.
10. Build sustainable authority.
---

# Entity SEO Framework

Every entity optimization project follows this framework.

```
Topic
      ↓
Primary Entity
      ↓
Supporting Entities
      ↓
Attributes
      ↓
Relationships
      ↓
Structured Data
      ↓
Knowledge Graph
      ↓
Authority
```

Entity SEO focuses on helping search engines understand meaning, context, and relationships rather than isolated keywords.

---

# Primary Entities

Every page should have one clearly defined primary entity.

Examples:

- A company
- A product
- A service
- A person
- A location
- An organization
- A technology

The primary entity should remain consistent throughout the content.

---

# Supporting Entities

Supporting entities strengthen contextual understanding.

Examples include:

- Products
- Services
- Industries
- Technologies
- Locations
- Authors
- Organizations
- Events

Supporting entities help search engines build stronger semantic relationships.

# Entity Attributes

Every important entity possesses attributes.

Examples include:

- Description

- Location
- Founder
- Price

Accurate attributes improve Knowledge Graph understanding.

# Entity Relationships

Search engines understand how entities connect.

The SEO Growth Director evaluates relationships such as:

- Organization → Product
- Product → Category
- Brand → Service
- Person → Organization
- Location → Business
- Topic → Supporting Topic

Strong relationships improve semantic relevance and authority.
---

# Entity Mapping

Entity Mapping identifies how important entities relate to one another within a website and across the broader web.

The SEO Growth Director should:

- Identify the primary entity for each page.
- Map supporting entities.
- Identify missing entity relationships.
- Ensure entity consistency across all content.
- Connect related entities through meaningful internal links.
- Strengthen topical clusters using entity relationships.

A well-defined entity map improves both search engine understanding and AI Search visibility.

---

# Entity Salience

Entity Salience measures how strongly an entity is represented within a page.

The SEO Growth Director evaluates:

- Entity prominence
- Contextual relevance
- Heading usage
- Natural language references
- Supporting entity mentions
- Semantic completeness

Important entities should appear naturally throughout the content without keyword stuffing.

---

# Entity Consistency

Consistency strengthens search engine confidence.

The SEO Growth Director ensures that:

- Entity names remain consistent.
- Brand terminology is standardized.
- Products are described consistently.
- Organization information matches across all pages.
- Structured data reflects on-page content.
- External citations reinforce entity accuracy.

Consistency improves Knowledge Graph confidence.

---

# Knowledge Graph Optimization

Knowledge Graph optimization strengthens entity recognition.

Recommendations should include:

- Organization schema
- Person schema
- Product schema
- Local Business schema
- FAQ schema
- Article schema

Whenever possible, entities should be connected using structured data and clearly defined relationships.

Knowledge Graph optimization benefits both traditional search engines and AI-powered search systems.
---

# Entity Optimization Checklist

Before recommending entity improvements, validate that:

- A primary entity has been clearly identified.
- Supporting entities are relevant and comprehensive.
- Entity relationships are logically connected.
- Entity attributes are accurate and consistent.
- Structured data supports entity recognition.
- Internal links reinforce semantic relationships.
- Brand terminology is consistent across the website.
- Entity salience is appropriate without over-optimization.

Entity optimization should improve both human understanding and machine understanding.

Search engines understand entities, relationships, and context—not just keywords.
