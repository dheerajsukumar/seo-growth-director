# Core Web Vitals

## Purpose

The Core Web Vitals knowledge base enables the SEO Growth Director to evaluate and improve real-world user experience through measurable performance metrics.

The objective is to enhance page experience, improve search visibility, reduce user frustration, and support long-term organic growth by optimizing website performance.

---

# Core Philosophy

Website performance is a user experience issue before it is an SEO issue.

The SEO Growth Director should recommend improvements that benefit both users and search engines by making websites:

- Faster
- More responsive
- More stable
- More accessible
- More reliable

Performance optimization should always balance user experience, technical feasibility, and business priorities.

---

The SEO Growth Director applies Core Web Vitals optimization to:

1. Improve page loading performance.

3. Reduce layout instability.
4. Improve interaction responsiveness.
5. Increase mobile usability.
6. Improve search performance.
7. Reduce abandonment.

# Largest Contentful Paint (LCP)

Largest Contentful Paint measures loading performance.

Recommended Target:

- Good: ≤ 2.5 seconds
- Needs Improvement: 2.5–4.0 seconds
- Poor: > 4.0 seconds

Common causes of poor LCP include:

- Slow server response
- Render-blocking CSS or JavaScript
- Large images
- Unoptimized fonts
- Excessive third-party scripts

Recommendations should prioritize improvements that reduce the time required to display the main page content.

---

# Interaction to Next Paint (INP)

Interaction to Next Paint measures overall responsiveness.

Recommended Target:

- Good: ≤ 200 ms
- Needs Improvement: 200–500 ms
- Poor: > 500 ms

The SEO Growth Director evaluates:

- JavaScript execution time
- Event handling
- Main-thread blocking
- Third-party scripts
- User interaction delays

Recommendations should improve responsiveness during real user interactions.

---

# Cumulative Layout Shift (CLS)

Cumulative Layout Shift measures visual stability.

Recommended Target:

- Good: ≤ 0.10
- Needs Improvement: 0.10–0.25
- Poor: > 0.25

Common causes include:

- Images without dimensions
- Ads inserted dynamically
- Font loading issues
- Content injected above existing elements
- Layout changes during page load

The objective is to ensure a stable and predictable user experience.
---

# Performance Diagnostics

Before recommending performance improvements, the SEO Growth Director should evaluate:

- Real User Monitoring (RUM)
- Lighthouse reports
- Core Web Vitals assessment
- Mobile performance
- Desktop performance
- Page resource loading
- JavaScript execution
- CSS optimization
- Third-party scripts
- Server response times

Recommendations should be based on measurable evidence rather than assumptions.

---

# Common Performance Bottlenecks

The SEO Growth Director should identify common issues such as:

- Slow hosting infrastructure
- Large image files
- Unoptimized JavaScript
- Render-blocking CSS
- Excessive third-party scripts
- Missing browser caching
- Poor CDN configuration
- Excessive DOM size
- Inefficient font loading
- Large video assets

Each bottleneck should be prioritized according to its impact on user experience.

---

# Optimization Priorities

Performance improvements should generally follow this order:

1. Improve server response time.
2. Optimize images.
3. Minify and defer JavaScript.
4. Optimize CSS delivery.
5. Reduce third-party scripts.
6. Implement browser caching.
7. Use a Content Delivery Network (CDN).
8. Optimize fonts.
9. Improve lazy loading.
10. Continuously monitor performance.

Recommendations should prioritize high-impact improvements before minor optimizations.

---

# Mobile Performance

Mobile users frequently experience slower networks and less powerful devices.

The SEO Growth Director should prioritize:

- Responsive design
- Mobile-first optimization
- Touch responsiveness
- Efficient image delivery
- Reduced JavaScript execution
- Stable layouts
- Fast interaction times

Performance recommendations should always consider the mobile user experience first.

---

# Performance Monitoring

Performance optimization is an ongoing process.

The SEO Growth Director recommends monitoring:

- Core Web Vitals
- Lighthouse scores
- PageSpeed Insights
- Google Search Console
- Real User Monitoring (RUM)
- Server performance
- Resource loading
- Error rates

Continuous monitoring helps identify regressions before they impact users.
---

# Core Web Vitals Optimization Checklist

Before recommending Core Web Vitals improvements, validate that:

- Largest Contentful Paint (LCP) meets Google's recommended threshold.
- Interaction to Next Paint (INP) is responsive across common user interactions.
- Cumulative Layout Shift (CLS) remains within acceptable limits.
- Mobile performance has been evaluated.
- Images are optimized and appropriately sized.
- JavaScript execution is efficient.
- CSS delivery has been optimized.
- Third-party scripts have been reviewed.
- Browser caching and CDN opportunities have been evaluated.
- Performance has been validated using both laboratory and real-user data.

Performance optimization should improve both user experience and business outcomes.
