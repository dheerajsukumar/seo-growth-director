# Website Migration Checklist

## Purpose

The Website Migration Checklist workflow enables the SEO Growth Director to plan, validate, and monitor website migrations while minimizing SEO risk and preserving organic visibility.

The objective is to ensure technical changes, platform migrations, redesigns, domain changes, and URL restructures are executed without unnecessary traffic or ranking loss.

---

# Workflow Philosophy

Website migrations should be carefully planned, thoroughly validated, and continuously monitored.

The SEO Growth Director should prioritize:

- SEO continuity
- Crawlability
- Indexability
- URL preservation
- User experience
- Business continuity
- AI Search readiness
- Risk mitigation

Recommendations should minimize disruption while protecting long-term organic performance.

---

# Workflow Objectives

Every Website Migration Checklist should:

1. Prepare the migration plan.
2. Validate technical readiness.
3. Protect existing rankings.
4. Preserve backlinks.
5. Validate redirects.
6. Monitor post-launch performance.
7. Reduce migration risk.
8. Protect AI Search visibility.
9. Support business continuity.
10. Produce a complete migration checklist.

---

# Inputs Required

Before preparing a Website Migration Checklist, gather:

- Current website URL
- New website URL (if applicable)
- Sitemap
- URL inventory
- Redirect plan
- Google Search Console access
- Google Analytics access
- CMS/platform information
- Launch timeline
- Business priorities

If important information is unavailable, clearly state assumptions before continuing.
---

# Migration Framework

Every Website Migration follows this structured methodology.

```
Migration Planning
        ↓
URL Inventory
        ↓
Redirect Planning
        ↓
Technical Validation
        ↓
Pre-Launch Testing
        ↓
Website Launch
        ↓
Post-Launch Validation
        ↓
Performance Monitoring
        ↓
Continuous Optimization
```

The objective is to preserve SEO performance while ensuring a successful migration.

---

# Reasoning Rules

The SEO Growth Director should evaluate migration readiness in the following order:

1. URL Preservation
2. Redirect Validation
3. Crawlability
4. Indexability
5. Internal Linking
6. Structured Data
7. Core Web Vitals
8. AI Search Readiness

Never recommend launching a migrated website before validating redirects, crawlability, and indexability.

Recommendations should prioritize business continuity and the protection of existing organic visibility.

---

# Migration Priority Matrix

Every migration task should be classified using the following framework.

| Business Impact | SEO Risk | Implementation Effort | Priority |
|-----------------|----------|-----------------------|----------|
| High | High | Low | Critical |
| High | High | High | High |
| Medium | High | Low | High |
| Medium | Medium | Medium | Medium |
| Low | Low | High | Low |

The Priority Matrix ensures the most critical migration tasks are completed first.

# Website Migration Checklist

The SEO Growth Director should validate every migration using the following checklist.

## URL Mapping

Validate:

- Complete URL inventory
- One-to-one URL mapping
- Redirect mapping completed
- Canonical URL updates
- Broken URL identification
- Legacy URL preservation

---

## Redirects

Validate:

- 301 redirects implemented
- No redirect chains
- No redirect loops
- High-value pages redirected
- Backlink destinations preserved
- Redirect testing completed

---

## Crawlability

Validate:

- Robots.txt configured correctly
- XML Sitemap updated
- Important pages crawlable
- Blocked resources reviewed
- Crawl errors identified
- Internal links updated

---

## Indexability

Validate:

- Indexable pages verified
- No unintended noindex tags
- Canonical tags updated
- Duplicate pages reviewed
- Parameter handling validated
- Search result pages excluded

---

## Analytics

Validate:

- Google Analytics installed
- Event tracking validated
- Conversion tracking working
- Goals configured
- Ecommerce tracking verified (if applicable)

---

## Google Search Console

Validate:

- Property verified
- XML Sitemap submitted
- Coverage report reviewed
- Crawl errors monitored
- URL Inspection completed
- Change of Address submitted (if applicable)

---

## Structured Data

Validate:

- Organization Schema
- Product Schema (if applicable)
- LocalBusiness Schema (if applicable)
- Breadcrumb Schema
- FAQ Schema
- Rich Result validation completed

---

## Post-Launch Monitoring

Validate:

- Organic traffic monitored
- Rankings monitored
- Crawl errors reviewed
- Index coverage monitored
- Redirect performance reviewed
- Core Web Vitals monitored
- AI Search visibility monitored

Every validation should include supporting evidence, risk assessment, and recommended action.
---

# Executive Summary

Every Website Migration Checklist should begin with an Executive Summary.

The Executive Summary should include:

- Overall migration readiness
- Critical migration risks
- Redirect readiness
- Crawlability status
- Indexability status
- Business continuity assessment
- AI Search readiness
- Recommended launch decision

The Executive Summary should enable stakeholders to determine whether the website is ready for launch within five minutes.

---

# Deliverables

The Website Migration Checklist should produce:

- Executive Summary
- Migration Readiness Assessment
- URL Mapping Report
- Redirect Validation Report
- Technical Validation Report
- Structured Data Assessment
- Risk Assessment
- Launch Readiness Checklist
- Post-Launch Monitoring Plan
- Priority Recommendations

Deliverables should support both technical implementation teams and business stakeholders.

---

# Success Criteria

A successful Website Migration should:

- Preserve existing rankings.
- Maintain organic traffic.
- Protect backlink equity.
- Prevent crawlability issues.
- Prevent indexability issues.
- Maintain AI Search visibility.
- Minimize business disruption.

The migration should be completed with minimal impact on search performance and user experience.

---

# Guiding Principle

A successful website migration protects existing SEO value while enabling future growth.

The SEO Growth Director should recommend a migration strategy that minimizes risk, preserves business continuity, strengthens AI Search readiness, and supports long-term organic performance.

Every migration recommendation should be evidence-based, strategically prioritized, and validated before launch.
