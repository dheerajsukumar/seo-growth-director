# Troubleshooting Guide

## Purpose

This guide helps diagnose and resolve common issues encountered while using the SEO Growth Director.

The objective is to maintain consistent, reliable, and predictable behavior across all workflows.

---

# Common Issues

## Issue 1 — Generic Recommendations

### Symptoms

- Recommendations lack specificity.
- Output feels generic.
- Business context is missing.

### Possible Causes

- Missing business objectives.
- Insufficient website information.
- Limited SEO data.

### Resolution

Provide:

- Business goals
- Website URL
- Industry
- Competitors
- Available SEO data

---

## Issue 2 — Conflicting Recommendations

### Symptoms

Different reports recommend different priorities.

### Possible Causes

- Different business assumptions.
- Conflicting datasets.
- Multiple objectives.

### Resolution

Reconfirm:

- Primary business objective
- Success metrics
- Available evidence

---

## Issue 3 — AI Search Recommendations Missing

### Symptoms

Traditional SEO recommendations only.

### Possible Causes

- AI Search workflow not triggered.
- No AI Search request.
- Limited entity information.

### Resolution

Explicitly request:

- AI Search Assessment
- Entity Optimization
- GEO Analysis
- LLM Visibility Review

---

## Issue 4 — Weak Executive Summary

### Resolution

Ensure:

- Business outcomes first
- Risks summarized
- Opportunities prioritized
- Recommendations actionable

---

## Issue 5 — Incomplete Reports

Verify:

- Correct workflow selected.
- Appropriate template applied.
- Required inputs available.
- Quality checklist completed.

---

# Diagnostic Checklist

Before investigating further verify:

- Repository structure unchanged.
- Latest documentation installed.
- Tests passed.
- Templates synchronized.
- Workflows updated.

---

# Escalation

If problems persist:

1. Review documentation.
2. Validate repository integrity.
3. Run integration tests.
4. Review CHANGELOG.
5. Update documentation if necessary.

---

# Guiding Principle

Most issues originate from incomplete context rather than incorrect reasoning.

Always validate inputs before modifying the system.
